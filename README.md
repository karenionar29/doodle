# 🎮 家庭遊戲間

小孩（George 小一、Win 小三）+ 爸爸的 AI 家庭遊戲收藏。
全部是**單檔 HTML、瀏覽器即玩、AI 在裝置裡跑、畫面聲音不外傳**。

- **玩：** https://karenionar29.github.io/doodle/
- **上傳檔案：** https://github.com/karenionar29/doodle/upload/main
- 目標：一千個遊戲 😄

---

## 📁 檔案規則

| 檔案 | 用途 |
|---|---|
| `index.html` | 遊戲大廳：先選「今天誰玩」→ 遊戲清單 + 評分系統 |
| `family-players.js` | **共用系統**：玩家名單/難度/性別 + 全域縮放鎖 |
| `gNN-名字.html` | 每款遊戲一個檔（g01-doodle、g02-hunt…） |

新遊戲 = 加一個 `gNN-xxx.html` + 在 `index.html` 的 GAMES 清單加一行。

## 🔧 每款遊戲都要遵守的規範

1. **玩家不寫死**：引入 `family-players.js?v=N`，用 `FP.get()` 拿名單、`FP.secs()` 算時間、依 `level` 調判定寬鬆度
2. **畫圖遊戲標配**：13 色、三段筆粗、🧽 擦子、↩️ 上一步（筆畫紀錄+重播架構）
3. **縮放鎖**：由 family-players.js 自動生效（小孩雙指誤觸不會把畫面弄跑）
4. **AI 推論一律在 Web Worker**，不卡畫面
5. **塗鴉模型（QuickDraw）要黑底白線**：AI 畫布保持透明、從 alpha 通道取值、邊界框裁切放大
6. **題庫防呆**：啟動時跟「模型實際認得的標籤」取交集，寫錯的題目自動剔除
7. **音效用 Web Audio 合成**，零音檔
8. CONFIG 物件集中所有可調參數

## 📤 交檔固定流程

1. 從 Claude 下載檔案 → 存到「檔案」App
2. 開上傳網址 → **一次傳一個** → Commit changes
3. 等 1 分鐘 → 開玩的網址，**後面加 `?v=數字` 破快取**

## ⚠️ 常見坑（別再踩）

- **Claude App 的預覽視窗測不了**（沒網路、找不到隔壁檔案）→ 一定要傳上 GitHub 用真網址測
- 開起來是舊的 → 快取，網址加 `?v=數字`
- 404 → 檔案沒傳上去，去 repo 檢查清單
- 共用 js 改版時，所有 html 的 `?v=` 要一起升
- 評分/玩家設定存在**各自裝置**的瀏覽器裡，Claude 看不到 → 用大廳的「📋 複製給 Claude」貼過去
